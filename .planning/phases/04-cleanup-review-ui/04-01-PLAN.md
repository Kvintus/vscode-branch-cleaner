## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01 | Elevation of privilege (user harm) | QuickPick acceptance | mitigate | Do not call `repository.deleteBranch` or any delete API in Phase 4; document return value handling so Phase 5 owns mutations. |
| T-04-02 | Spoofing / misleading trust | Row `detail` / icons vs actual merge | mitigate | Build `detail` strings only from `row.merge` + `plan.baseline.displayLabel`; no alternate baseline strings in UI layer (D-02 / DOM-04 alignment). |
| T-04-03 | Information disclosure | Title/placeholder | accept | Baseline ref labels are repo-local metadata already exposed in SCM; no secrets (ASVS L1: no credentials in UI). |
| T-04-04 | Tampering | Branch name in pick | accept | Names originate from `vscode.git` / plan builder, not user-authored picker input. |
| T-04-05 | Denial of service | Large branch lists | accept | Same scale as Phase 3 enumeration; no new network or unbounded loops in UI path. |

Command injection: **N/A** (no shell from picker values in Phase 4).
